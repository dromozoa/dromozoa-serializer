テーブルの等価

テーブル参照のキーが存在し、テーブル参照が循環する場合、等価性は容易に判定できない。 そこで、テーブルを有向グラフととらえて、グラフ同型ならば等価であるとみなすことにする。

この規則のもとでは、次のテーブルaとbは等価でない。 

local a = { { 42 }, { 42 } }

local t = { 42 }
local b = { t, t }

グラフ同型の判定のための良く知られたアルゴリズムが存在する。 しかし、

グラフ同型の判定

A Fast Backtracking Algorithm to Test Directed Graphs for Isomorphism Using Distance Matrices

Algorithm 2: Backgracking Algorithm to Test \(G^1\) and \(G^2\) for Isomorphism

  1. \(t = 0\)と設定し、 \(p_i = 0\) \((0 \le i \le N)\)と設定する （\(p_i\)はレベル\(t\)で選ばれたグラフ\(G^1\)の頂点）。
  2. \(C^1_0\)と\(C^2_0\)をアルゴリズム1で得られたクラスベクトルとし、 \(K^1_t\)と\(K^2_t\)をクラスカウントベクトルとする。
  3. \(t = N\)ならば、グラフは同型であると結論し、 マッピングを提示して終了する。
  4. \(i = p_t\)と設定する。
  5. \(i \ne 0\)ならば、7へ。
  6. \(v^1_i\)が低いレベルでマップされていないような整数\(i\)を択ぶ。 一意なマップされていない頂点\(c^i_{rt}\)が存在するならば、 \(p_t = i\)と設定する。
  7. \(c^1_{it} = c^2_{rt}\)となり、 \(v^2_r\)のために択ばれたマッピングが存在しないように 整数\(r\)を択ぶ。
  8. そのような\(r\)が存在したら、10へ。
  9. \(t\)を1減らす。 \(t \lt 0\)ならば、グラフは同型でないと結論する。 そうでなければ、7へ。
  10. \(C^1_t\)と\(D^1\)の行\(i\)を合成して\(\hat{C^1}\)を得る。 \(C^2_t\)と\(D^2\)の行\(r\)を合成して\(\hat{C^2}\)を得る。 \(\hat{C^1}\)と\(\hat{C^2}\)のそれぞれの一意項について、 一意な整数を置換することで\(\tilde{C^1}\)と\(\tilde{C^2}\)を生成する。 \(\tilde{C^1_t}\)と\(D^1\)の行\(i\)を合成して\(\dot{C^1}\)を得る。 \(\tilde{C^2_t}\)と\(D^2\)の行\(r\)を合成して\(\dot{C^2}\)を得る。 \(\dot{C^1}\)と\(\dot{C^2}\)のそれぞれの一意項について、 一意な整数を置換することで\(C^1_{t+1}\)と\(C^2_{t+1}\)を生成する。
  11. クラスカウントベクトル\(K^1_{t+1}\)と\(K^2_{t+1}\)を計算する。
  12. \(K^1_{t+1} \ne K^2_{t+1}\)ならば、7へ。
  13. \(t\)を1増やす。
  14. 3へ。

local b = { [a] = 42 }
a[b] = 42

c[c] = 42

assert(equiv(a, b))
assert(not equiv(a, c))

まず、テーブル参照のキーを無視して判定を行う。 これにより、

テーブルをグラフ構造としてとらえて、グラフ同型ならばふたつのテーブルを等価であるとみなすことにする。 それぞれのグラフについて、ただひとつの始点ノードがわかっているため、一般的な判定問題よりもやさしい（と思う）。

テーブル参照をキーに持たない場合、キーは順序づけ可能なので容易に解くことができる。 テーブル参照をキーに持つ場合、テーブル参照が循環している場合を考慮すると、グラフ同型かどうかを判定する必要がある。

まず、単純なはなしとして、

深さ優先探索により、グラフをtree edgeで構成されるspanning treeとback edgeとcross or forward edgeに分解する。spanning treeが等価でなければグラフは等価でない。

テーブルが以下の条件を満たすとき、再帰を用いて容易にテーブルを比較できる。

  1. テーブル参照をキーに持たない。
  2. テーブル参照が循環していない。

テーブル参照をキーに持つとき、等価な複数のテーブル参照がキーに含まれているかもしれない。つまり、 

local x = {
  [{42}] = 17;
  [{42}] = 23;
}

テーブル参照が循環しているとき、グラフが同型かどうかを判別する。次のテーブルは等価でない。 

local y = {}
local x = { foo = y }
y.foo = x

local z = {}
local y = { foo = z }
local x = { foo = y }
z.foo = x

深さ優先探索でバックエッジが存在するかどうかを調べることで、テーブル参照の循環を検出する。

下記のふたつを等価とみなすかどうかを検討する必要がある。単純な再帰による比較では等価と見なされるが、グラフが同型でない。 

local y = { 42 }
local x = { y, y }

local x = { { 42 }, { 42 } }
