テーブルの等価とグラフ同型

テーブルをグラフ構造としてとらえたとき、グラフ同型ならばふたつのテーブルは等価であるとみなす。 それぞれのグラフについて、ただひとつの始点ノードがわかっているため、一般的な判定問題よりもやさしい（と思う）。

テーブル参照をキーに持たない場合、キーは順序づけ可能なので問題は易しい。 そのため、テーブル参照をキーに持つ場合を考える。

まず、単純なはなしとして、

深さ優先探索により、グラフをtree edgeで構成されるspanning treeとback edgeとcross or forward edgeに分解する。spanning treeが等価でなければグラフは等価でない。

テーブルが以下の条件を満たすとき、再帰を用いて容易にテーブルを比較できる。

  1. テーブル参照をキーに持たない。
  2. テーブル参照が循環していない。

テーブル参照をキーに持つとき、等価な複数のテーブル参照がキーに含まれているかもしれない。つまり、 

local x = {
  [{42}] = 17;
  [{42}] = 23;
}

テーブル参照が循環しているとき、グラフが同型かどうかを判別する。次のテーブルは等価でない。 

local y = {}
local x = { foo = y }
y.foo = x

local z = {}
local y = { foo = z }
local x = { foo = y }
z.foo = x

深さ優先探索でバックエッジが存在するかどうかを調べることで、テーブル参照の循環を検出する。

下記のふたつを等価とみなすかどうかを検討する必要がある。単純な再帰による比較では等価と見なされるが、グラフが同型でない。 

local y = { 42 }
local x = { y, y }

local x = { { 42 }, { 42 } }
